# HTTP

### 1. Definition

HTTP (HyperText Transfer Protocol) is a protocol used to send and receive data between a client and a server.

### 2. Why it is used

It helps browsers and applications communicate with web servers.

### 3. How it works

Client sends an HTTP request → Server processes it → Server sends an HTTP response.

### 4. Real-time Example

When you open a website or use a REST API, HTTP is used to transfer data.

### 5. Request Structure

* Method (GET, POST, PUT, DELETE)
* URL
* Headers
* Body (optional)

### 6. Response Structure

* Status Code
* Headers
* Body

### 7. Common Status Codes

* 200 - OK
* 201 - Created
* 400 - Bad Request
* 401 - Unauthorized
* 404 - Not Found
* 500 - Internal Server Error

### 8. Advantages

* Simple and fast
* Platform independent
* Supports REST APIs

### 9. Disadvantages

* Stateless
* HTTP is not encrypted (HTTPS is secure)

### 10. Best Practices

* Use correct HTTP methods.
* Return proper status codes.
* Use HTTPS for security.

### 11. Interview Questions

* What is HTTP?
* What is the difference between HTTP and HTTPS?
* Explain the request-response cycle.
* What are common HTTP status codes?

### 12. Dry Run

Client → HTTP Request → Server → HTTP Response → Client

### 13. Industry Usage

HTTP is used in websites, mobile apps, REST APIs, and microservices for communication.
